# UI Skill

This skill provides comprehensive WC Design: Accessibility AuditAG compliance auditing for UI code and interfaces.

## Capabilities

### Accessibility Auditing
- Conduct WCAG 2.2 accessibility audits
- Identify and document accessibility violations
- Provide remediation guidance for common issues
- Check for color contrast, focus management, and keyboard navigation

### Automated Testing
- Run automated accessibility tests
- Use axe-core and other accessibility testing tools
- Generate accessibility reports
- Prioritize issues by severity

### Manual Verification
- Verify screen reader compatibility
- Check keyboard navigation patterns
- Validate ARIA implementation
- Test with assistive technologies

### Remediation Support
- Provide fixes for common WCAG violations
- Suggest accessible design patterns
- Guide implementation of ARIA attributes
- Recommend semantic HTML improvements

## Usage Examples

- "Audit this UI for WCAG compliance"
- "Fix the accessibility issues in this component"
- "Check if this page meets WCAG 2.2 AA standards"
- "What accessibility issues exist in this code?"
- "Help me make this button keyboard accessible"

## Prerequisites

This skill works with any UI codebase. For automated testing, ensure appropriate testing libraries are installed in the project.
